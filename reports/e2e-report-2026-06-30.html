Executive summary

The client-reported bug — Best selling on the KEYTAG category showed low-ID products instead of true best sellers (e.g. Psyduck on page 6) — is resolved. Elementor Loop Grid Current Query now receives WooCommerce SQL ordering before pagination.

  • Root cause: Elementor secondary query used suppress_filters and bypassed posts_clauses; v1.3.5’s late wp hook ran after the main query executed.
  • Fix (v1.3.6): Force suppress_filters = false, hook elementor/query/{id}, register posts_clauses on init when ?orderby= is present.
